## Projects Active This Week

### project-norma (Experiment) - Active

**What it is:** A voice memo app that lets you toggle between dictation mode and assistant mode mid-recording. Imagine recording your thoughts, then pressing a button to ask Claude a question or request a lookup, then switching back to dictation - all in one continuous recording. The transcript later shows your dictation with Claude's responses woven inline.

**Where you're at:** Early prototype stage. You've validated the core concept and built working pieces. Running a bit out of steam but made major progress.

**This week's journey:** You started by getting Whisper running locally and testing it with real voice memos. First discovery: the base model is way better than the tiny model at understanding context - it correctly transcribed "limbic capitalism" where tiny mangled it as "limpy capitalism." The extra processing time (38 seconds vs 25 seconds for a 2.5 minute clip) is totally worth it for quality.

Then you tested whether Whisper's `initial_prompt` parameter would help - maybe feeding it previous segments would improve accuracy? The experiment was beautifully designed (three audio segments simulating the dictation → assistant → dictation flow), but the result was surprising: **zero impact**. Character-for-character identical transcripts whether you provided context or not. This felt like a dead end until...

**The breakthrough:** Through the failed prompt experiment, you realized you'd been thinking about the problem wrong. Whisper isn't supposed to generate responses. It's not an assistant. It just transcribes speech to text. The actual assistant functionality - answering questions, looking things up, giving advice - that's a **second stage**.

This is the "aha moment" that reframed everything: Stage 1 is transcription (Whisper converts audio → text with mode labels). Stage 2 is consolidation (Claude processes those transcripts and generates actual responses for the assistant segments). You're not trying to make Whisper smart - you're using it as a dumb transcriber and letting Claude be the smart one.

You built a consolidation prototype to test this. Fed it three segments from your tests, and Claude generated a perfect inline response: contextually aware, practical advice, concise, naturally flowing. It just worked.

Then you tackled the recorder piece. You found a minimal 24-line Python audio recorder on GitHub and got it running. Hit a mono/stereo channel mismatch (MacBook mic is mono, script assumed stereo), fixed it in one line, and had a working foundation.

You also made a smart UX decision: skip the "companion tracker app" idea. Originally you thought about building a separate timestamp tracker that runs alongside Voice Memos, but you realized juggling two apps would break flow and create timing sync nightmares. Better to build one integrated app that handles both recording and mode switching.

**What's next:** The path forward is clear - build out the integrated recorder with mode toggle buttons and timestamp tracking. You've proven the core concept works. The prototype exists. Now it's about connecting the pieces.
